## 2. 分析すること

### 2.1 定量分析

以下を件数・率で集計してください。

```txt
総架電数
受付突破数 / 受付突破率
本人接続数 / 本人接続率
部署特定数 / 部署特定率
関心軸取得数 / 関心軸取得率
資料送付数 / 資料送付率
再連絡許可数 / 再連絡許可率
企画骨子案打診数 / 企画骨子案打診率
二次商談化数 / 二次商談化率
停止数 / 停止率
```

### 2.2 温度分類

各ログを以下に分類してください。

| 分類 | 状態 | 次アクション |
|---|---|---|
| A：企画骨子案OK | 用途が明確で15〜20分の場が取れる | 二次商談化 |
| B：資料送付後フォロー | 興味はあるが即日程化しない | 資料請求フォロー台本へ接続 |
| C：担当部署確認 | 本人・受付が部署を探している | 部署確認後に再架電 |
| D：時期フォロー | 予算・来期施策・タイミング待ち | 時期指定フォロー |
| E：停止 | 用途なし・拒否強い・再連絡不可 | 深追い停止 |

### 2.3 関心軸

以下に分類してください。

```txt
採用広報
コーポレート広報 / PR
営業前の会社紹介
経営者・事業責任者発信
来期施策 / 予算検討
不明
```

### 2.4 誤認回収

以下の誤認がどこで起きたかを抜き出してください。

```txt
YouTube広告だと思われた
動画制作だと思われた
広告枠販売だと思われた
社長出演必須だと思われた
費用が重い施策だと思われた
```

誤認が多い場合は、どの表現が原因になったかを推定し、受付突破トーク・本人接続トークの修正案を出してください。

### 2.5 ログ引用

重要なログは短く引用してください。
ただし、個人名・電話番号・メールアドレスは伏せる。

引用する観点：

```txt
受付突破できた言葉
受付で止まった言葉
本人が反応した言葉
資料送付で止まった理由
再連絡許可が取れた言い回し
二次商談化できた言い回し
深追いすべきでない拒否パターン
```

## 4. 判断ルール

- アポ数だけで良し悪しを判断しない。
- 受付突破・部署特定・関心軸取得・再連絡許可も成果として見る。
- SPを否定しない。通常架電の結果から、リスト・トーク・架電者スキルを分けて判断する。
- 不明点は `{未確認}` と書く。
- 顧客向け資料に使う場合は、内部評価・営業側の弱み・AI作業メモを消す。
