# pain_to_solution_flow｜課題整理型 商談プレゼン構成

## 用途

見込み客向けの商談プレゼンで、いきなりサービス説明をせず、相手の課題整理から提案へ接続する型。

## 向いている場面

- 相手がまだ課題を明確に言語化できていない
- サービス説明から入ると売り込み感が出る
- 既存取引先や既存業務を否定したくない
- 決裁者へ社内共有しやすい資料にしたい

## 基本構成

```txt
1. 表紙
2. 前回商談の整理
3. すでに見えている課題
4. まだ見えていない論点
5. 放置した場合の負担
6. 提案の位置付け
7. 小さく始める方法
8. 次回アクション
```

## コア思想

```txt
売る前に、相手の頭の中を整理する。
```

## スライド別の役割

### 1. 表紙

相手名・目的・日付を明確にする。

### 2. 前回商談の整理

「こちらは話を理解している」と示す。

### 3. すでに見えている課題

相手が自覚している課題を言語化する。

### 4. まだ見えていない論点

提案につながる未整理領域を提示する。

### 5. 放置した場合の負担

煽りではなく、判断材料として整理する。

### 6. 提案の位置付け

サービス説明ではなく「この課題に対する打ち手」として接続する。

### 7. 小さく始める方法

導入ハードルを下げる。

### 8. 次回アクション

相手が次に何をすればよいか明確にする。

## 禁止

- 2枚目からサービス説明に入る
- 課題を決めつける
- 相手の既存体制を否定する
- 価格だけで判断させる

## 使える一言

```txt
仕入先変更ではなく、困った時の選択肢を一枚増やすイメージです。
```

```txt
まずは御社側で判断しやすいよう、論点を整理させてください。
```
