# slides/patterns｜スライド構成パターン集

ここには、案件横断で再利用できるスライド構成パターンを置く。

## 目的

毎回ゼロからスライド構成を考えず、商談目的に応じて勝ち筋の型を選べるようにする。

```txt
商談ログ
↓
勝ち筋
↓
構成パターン選択
↓
個別案件向けに調整
↓
HTMLスライド化
```

## 初期パターン

```txt
pain_to_solution_flow.md       = 課題整理から提案へつなぐ
executive_decision_flow.md     = 役員判断用に短くまとめる
demo_bridge_flow.md            = デモ前に見るポイントを揃える
meeting_feedback_flow.md       = 商談FB/PM報告用
infographic_flow.md            = 複雑な仕組みを一枚で図解する
```

## パターン化する条件

以下のどれかを満たしたら、patternsへ抽出する。

- 2案件以上で使えそう
- 商談の反応が良かった
- 決裁者同席につながった
- 次回化につながった
- 説明が難しい商材を簡単に伝えられた
- PM報告や社内共有で再利用できそう

## 禁止

- 個別顧客名を含めたままパターン化しない
- 成功理由が不明なまま型にしない
- デザインだけを型にしない

## 一言

```txt
デザインパターンではなく、勝ち筋パターンを残す。
```
