# 株式会社センド｜ICT教材・社内掲載営業 補足OS

更新日：2026-06-30  
情報源：membersチャット履歴、Drive既存資料  
格納方針：原文ログ、個人情報、連絡先、文字起こし全文は格納しない。営業判断・運用ルールのみ記録する。

---

## 1. 結論

湘南ベースレインボー / ICT教材の企業向けルートは、当面以下で扱う。

```txt
企業に福利厚生として一括導入してもらう営業ではなく、
社員家族向けの学習支援情報として、社内掲示板・福利厚生案内・社内ポータル等に掲載してもらう営業。
```

初回ゴールは即商談・即導入ではなく、以下を基本にする。

```txt
資料送付
↓
コールバック了承
↓
社内掲示・福利厚生案内・社内ポータルへの掲載可否確認
```

---

## 2. 背景

membersチャット履歴上、直近のICT教材架電は、健康経営優良法人系の人事・総務リストに対して実施されている。  
また、運用上の着地目標は、アポイント直取りではなく「資料送付着地のコールバック了承」と整理されている。

そのため、商談化を急ぐよりも、まず人事・総務側に「社員向け情報として案内できるか」を確認する設計が現実的。

---

## 3. 刺さりやすい入口

強い入口は、長期休暇前後の家庭学習負担。

```txt
長期休暇の時期は、家庭側の学習サポート負担が増えやすい。
そのため、社員向けの支援情報として、学習支援サービスを社内掲示・福利厚生案内に掲載できないか相談する。
```

言い換え：

```txt
福利厚生制度を新しく導入してください、ではなく、
社員向けの情報提供として掲載いただけないか、という相談。
```

---

## 4. 受付突破トーク

```txt
社員家族向けの家庭学習支援サービスについて、
長期休暇前後の社員向け情報としてご案内しておりまして、
人事・総務、または福利厚生/社内掲示のご担当者様に確認をお願いできますでしょうか。
```

---

## 5. 本人接続後トーク

```txt
今回は、会社様に福利厚生制度としてすぐ導入いただくというより、
社員向けの情報として、社内掲示板・福利厚生案内・社内ポータル等に
掲載いただけるかのご相談です。

まずは資料をご確認いただき、掲載可否や必要な資料形式について、
後日あらためて確認させていただければと思っております。
```

---

## 6. 避ける入口

```txt
・ICT教材を導入しませんか
・福利厚生サービスを契約しませんか
・離職率改善サービスです
・従業員満足度向上の教材です
```

理由：既に福利厚生が充実している企業、定着状況に大きな課題感がない企業では「現状不要」で止まりやすい。  
まずは「掲載相談」として軽く入る。

---

## 7. コールバック / 商談で確認する論点

```txt
・社内掲示板、社内ポータル、福利厚生案内、社内報などの掲載可否
・掲載判断者
・掲載に必要な資料形式
・社員個人負担か、会社負担か
・会社として取りまとめるのか、社員個人申込にするのか
・長期休暇前後の案内タイミング
・社員家族向け福利厚生の既存有無
・資料送付後のコールバック了承可否
・本格導入検討に進める条件
```

---

## 8. 営業マネのログ評価基準

アポ数だけで評価しない。以下を見る。

```txt
・長期休暇の家庭学習負担に反応したか
・社員家族向け福利厚生に反応したか
・会社負担か従業員負担かを確認できたか
・社内掲示板/社内ポータル/福利厚生案内の担当部署を確認できたか
・資料送付後のコールバック了承を取れているか
・導入検討ではなく、掲載相談として申し送りできているか
・資料請求の温度感を、高温/低温/判断不能で分けられているか
```

---

## 9. 営業マネの役割

```txt
架電者を詰めるのではなく、入口定義を揃える。
「福利厚生導入営業」ではなく「社員向け情報掲載営業」として、
受付突破・本人接続・資料送付後コールバック・商談申し送りを統一する。
```
