# ジェネレートワン｜本人資料請求フォロー架電台本

## 0. 台本の目的

この台本の目的は、資料送付後に「資料をご覧いただけましたか？」で終わらせることではありません。

目的は、本人資料請求者の関心軸を確認し、株式会社ジェネレートワンの「日本企業超会議」を、単なる動画制作ではなく、採用・営業・ブランディングに使える番組出演/コンテンツ資産として再定義し、次回の企画骨子案提示の場を設定することです。

ゴールは以下です。

```txt
本人資料請求
↓
資料送付後フォロー
↓
関心軸確認
↓
活用可能性の仮説提示
↓
企画骨子案の二次商談化
```

## 1. 冒頭トーク

「お世話になっております。先日、日本企業超会議の資料をお送りした件でご連絡しました、ジェネレートワンの〇〇です。今、1〜2分だけよろしいでしょうか。」

「先日資料をお送りした件でのご連絡です。資料の内容を一方的に説明したいというより、御社の場合だと、採用向け・営業向け・経営者ブランディング向けのどこが近そうかだけ、一度確認できればと思いまして。」

相手が忙しそうな場合：

「長くはお取りしません。資料をご覧いただいたかどうかより、御社で使うならどの方向が近そうかだけ確認できればと思っています。」

## 2. 資料確認の聞き方

### NG

「資料はご覧いただけましたか？」

これだけだと、見ていない場合に会話が終わります。見ていた場合も感想確認だけで終わり、次回商談に進みません。

### OK

「資料、もしまだ全部ご覧いただけていなくても大丈夫です。今回の話は、資料の細かい説明というより、御社で使うならどの用途が近いかを一緒に確認するイメージです。」

「ざっくりで構わないのですが、採用・営業先への認知・経営者や会社の見せ方、このあたりで少しでも近そうなものはありますか？」

見ている場合：

「ありがとうございます。資料の中で、御社に近そうだなと思った部分、逆に少し違うなと思った部分はありましたか？」

見ていない場合：

「お忙しいと思うので大丈夫です。資料だけだと一番大事な『御社ならどう使えるか』が伝わりにくいので、そこだけ確認できればと思っています。」

## 3. 関心軸の切り分け質問

「今回の日本企業超会議は、動画を作るだけというより、企業のリーダーや事業の魅力を番組として見せて、その後も採用・営業・ブランディングで使えるコンテンツ資産にするものです。」

「御社で考えると、近いのはどれでしょうか。」

1. 「採用で、会社の雰囲気や代表・責任者の考え方を伝えたい」
2. 「営業や認知拡大で、初回商談前に会社理解を進めたい」
3. 「経営者や会社の顔になる人を前に出して、信頼感を作りたい」
4. 「決算前や予算消化で、来期以降にも残る施策を検討したい」
5. 「今は具体的ではなく、まず情報収集に近い」

相手が答えにくそうな場合：

「どれか1つに決めなくて大丈夫です。強いて言うと、採用・営業・ブランディングの中だと、どれが一番近そうですか？」

## 4. 分岐A：採用ブランディングに関心

「採用寄りですね。ありがとうございます。」

「この企画は、求人広告のように条件だけを見せるというより、代表や責任者の考え方、会社が何を大事にしているかを番組として見せられるのが特徴です。」

「候補者からすると、入社前に会社の温度感やリーダーの考え方が分かるので、応募の質や志望度を上げる材料として使いやすいです。」

確認質問：

「今、採用でいうと、応募数を増やしたいのか、応募者の質を上げたいのか、あるいは入社後のミスマッチを減らしたいのか、どこが一番近いですか？」

二次商談への橋渡し：

「もし採用向けで考えるなら、御社の場合はどんな切り口で見せると候補者に刺さるか、簡単な企画骨子にしてお話しできます。」

## 5. 分岐B：営業・認知拡大に関心

「営業・認知拡大寄りですね。」

「この場合、単に動画を作るというより、第三者の番組に出る形で、御社の事業や考え方を分かりやすく見せるのがポイントです。」

「営業資料だけだと伝わりにくい会社の信頼感や、なぜこの事業をやっているのかを、商談前に見てもらえる資産にできます。」

確認質問：

「営業面で使うとしたら、初回商談前の信頼づくり、既存顧客への認知、展示会や紹介時の説明補助、このあたりだとどれが近そうですか？」

二次商談への橋渡し：

「御社の営業先に見せる前提で、どんな番組切り口にすると伝わりやすいか、一度たたき台をお持ちできます。」

## 6. 分岐C：経営者/会社の見せ方に関心

「経営者や会社の見せ方ですね。」

「ここは日本企業超会議とかなり相性がいいです。企業のリーダーや事業責任者を、単なるインタビューではなく、物語の主人公として見せる設計ができます。」

「なお、出演は必ず社長でないといけないわけではありません。責任者の方や、会社の顔として語れる方でも成立します。」

確認質問：

「御社で前に出すとしたら、代表の方、事業責任者の方、採用や営業の責任者の方だと、どなたが一番イメージに近いですか？」

二次商談への橋渡し：

「その方を前面に出すなら、どんなテーマで見せると御社らしさが伝わるか、次回は企画骨子案でお話しできます。」

## 7. 分岐D：予算消化・決算前施策に関心

「予算や時期の観点ですね。」

「この企画は、単発の広告費として消えるというより、撮影後も採用サイト・営業資料・SNS・商談前送付などに使えるコンテンツ資産として残せるのが特徴です。」

「決算前や余剰予算の活用としても、来期以降に残るものに変えやすいです。」

確認質問：

「もし検討するとしたら、今期中に使い切る予算の話なのか、来期施策として候補に入れる話なのか、どちらが近いですか？」

二次商談への橋渡し：

「予算化前に、御社だとどういう企画にできそうかだけ見ておくと、判断しやすいと思います。次回は一般説明ではなく、簡単な骨子案をお持ちします。」

## 8. 分岐E：資料請求が断り代替だった場合

相手：
「まだ見ていません」
「必要ならこちらから連絡します」
「今は特に考えていません」

切り返し：

「承知しました。売り込みたいというより、不要であれば不要で大丈夫です。」

「ただ、最後に1点だけ確認させてください。御社でこういう番組出演やコンテンツ資産を使うとしたら、採用・営業・ブランディングの中だと、どれが一番近そうですか？」

回答がある場合：

「ありがとうございます。では、その観点に絞って資料を見ると判断しやすいと思います。○日頃に一度だけ、その後のご感触を伺ってもよろしいですか？」

回答がない場合：

「承知しました。では、今回はいったん資料だけご確認いただければ大丈夫です。不要でしたらもちろん結構ですので、また時期が合いそうな際にご案内します。」

## 9. 二次商談への打診

「もし方向性が合いそうであれば、次回は一般的な説明ではなく、御社の場合どんな企画にできそうか、簡単な骨子案ベースでお話しできればと思うのですが、来週あたり15〜30分ほどお時間いただくことは可能でしょうか。」

日程打診：

「来週ですと、○曜日の午前か、○曜日の午後でしたらどちらがご都合よろしいですか？」

担当者同席を促す場合：

「もし採用や広報のご担当者様も関わるようでしたら、その方もご一緒の方が話が早いかもしれません。」

## 10. 断り・保留への切り返し

| 相手の反応 | 返し方 |
|---|---|
| まだ資料を見ていない | 「大丈夫です。資料の細かい説明ではなく、御社ならどの用途が近いかだけ確認できればと思っています」 |
| 興味があれば連絡する | 「ありがとうございます。では、こちらからも○日頃に一度だけ、その後のご状況を伺ってもよろしいですか」 |
| 担当者に確認する | 「ありがとうございます。では、ご担当者様が見る時に判断しやすいよう、採用/営業/ブランディングのどこで見るべきかだけ整理してお送りします」 |
| 予算がない | 「承知しました。今すぐ決める話ではなく、予算化前の判断材料として、御社ならどう使えるかだけ見ていただく場でも大丈夫です」 |
| 動画制作は間に合っている | 「ありがとうございます。動画制作というより、番組出演をきっかけに採用・営業・ブランディングで使える資産を作る話に近いです」 |
| 社長出演は難しい | 「社長でなくても大丈夫です。責任者の方や、会社の顔として語れる方でも成立します」 |
| 今は必要ない | 「承知しました。では時期が合う前提で、○月頃に一度だけ情報提供してもよろしいでしょうか」 |

## 11. NGトーク

- 「資料をご覧いただけましたか？」だけで終わる。
- 見ていない相手に、資料の内容を最初から説明し直す。
- 相手の関心軸を聞かずに、商材説明を続ける。
- 「動画制作です」と表現して、制作会社の相見積もり枠に入る。
- 社長出演が必須のように話す。
- 「興味があればご連絡ください」で終える。
- 予算・時期・決裁者・担当部署確認に踏み込まない。
- 断られた後に粘りすぎて、次回連絡の許可まで失う。

## 12. 架電者向け注意点

- 本人資料請求者は、新規架電より温度が高い。放置せず、資料送付後1週間以内を目安に追う。
- フォローの主語は「資料」ではなく「御社でどう使えそうか」。
- 相手が資料を見ていなくても、責めない。むしろ関心軸確認に切り替える。
- 採用・営業・ブランディング・予算のどれに近いかを必ず記録する。
- 二次商談の勝ち筋は「一般説明」ではなく「御社向け企画骨子案」。
- 個人名・電話番号・メールアドレスはGitHubに残さない。ログ原本はDriveを参照する。
