# GPTプロジェクトチャット整理プロトコル

## 目的
プロジェクトチャットを削除しても営業支援知見が失われないよう、必要情報だけをGitHub/Driveへ移す。

## 基本方針
チャット全文は保存しない。営業OSに再利用できる情報だけを抽出する。

## 分類

| 種別 | 保存先 | 内容 |
|---|---|---|
| 案件判断 | `clients/{社名}.md` | 商材再定義、ターゲットの歪み、架電/商談勝ち筋、進捗、次の一手 |
| 横断知見 | `patterns/*.md` | KPI分析、拒否分類、二次トスアップ、FBテンプレ、PM報告テンプレ |
| OS設計 | `docs/*.md` | Drive/GitHub/Codex/Vercel運用、自動化構想、構成ルール |
| AI指示文 | `prompts/*.md` | Claude/Gemini/Codex/GPTへの起動プロンプト |
| 原本 | Google Drive | 動画、文字起こし、顧客資料、CSV、Slides/Docs/PDF |
| 不要 | 保存しない | 雑談、愚痴、却下済み案、重複、短文返信 |

## チャットごとの処理手順
1. チャット名から案件/横断/OS/不要を判定する。
2. 「事実」「分析」「仮説」「未確認」を分けて抽出する。
3. 案件に紐づくものは該当カルテの2〜5章に移す。
4. 他案件でも使える型は `patterns/` に移す。
5. 運用ルールや自動化構想は `docs/` に移す。
6. AIへ再実行させたい指示文は `prompts/` に移す。
7. GitHubへcommit後、チャットを削除する。

## 保存判断の基準
残すべき情報：
- 次回の商談品質が上がる
- 台本修正に使える
- PM/商談者/架電者へのFBに使える
- 他案件へ横展開できる
- Codex/Claude/GPTに再実行させられる

残さない情報：
- 感情整理のみ
- その場限りの一文返信
- 既にカルテ/台本/資料に反映済みの重複
- 古い案で、明確に却下されたもの

## 例外
愚痴・トラブルでも、運用の型に昇華できるものは残す。

例：
- 架電者が台本を勝手に変えた → `patterns/架電者FBテンプレ.md`
- 引継ぎ責任の押し付けが発生 → `patterns/責任分界点_切り分けテンプレ.md`
- PMが販売を知らずに進めた → `docs/sm_abc_operation.md`
