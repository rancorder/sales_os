# _analysis_protocol.md｜営業OS 分析プロトコル

エージェント（Codex / Claude Code）が各社カルテを分析・更新するときの手順書。
**人間が読むためでなく、エージェントに食わせるための正本。**

---

## 0. 大原則（2層構造）

```
【入力｜一定期間分】          【処理｜各社・差分】       【出力｜全体総括】
商談動画の文字起こし  ┐                            ┌→ 架電台本修正の原材料
                     ├→ 各社カルテ4章に追記 ──→─┼→ 商談台本修正の原材料
架電ログ（SC定例）    ┘   （分析済みマーカー付き）  └→ 勝ちパターン抽出
```

- **個別処理は差分で回す**：既に分析済みの期間はスキップ。
- **総括は全部読む**：スキップした期間も、4章の本文は総括の材料に使う。

---

## 1. 入力の置き場ルール

### 架電ログ
以下のいずれかの形で各社フォルダ直下に置く（どちらでも認識する）：
- 期間フォルダ：`260601-260630_架電ログ/` または `260601-0630_架電ログ/`
- 親フォルダ集約：`架電ログ/` の直下に期間表示のファイル/サブフォルダ
- 実体はアイドマの **SU定例資料**（集計期間・電話数・着電率・各種率・本人拒否分類）と **SCデータ**（CSV/xlsx）。

### 商談文字起こし
- 各社フォルダの **`商談動画/`** フォルダ内に置く。
- 動画本体（mp4）は読めない。**文字起こし（.txt / Google Doc / 字幕）が分析対象**。
- 命名推奨：`260617_{相手企業名}_商談文字起こし`

### 期間の単位
- **週次**（SU定例の集計期間にそのまま乗せる。例：`260530-260617`）を基本とする。
- 月次でまとめたい時は月フォルダ（例：`260601-260630`）でも可。

---

## 2. 各社の処理手順（差分ループ）

各社について順に：

1. カルテ4章を読み、**既存の「分析済み」見出し**を集める。
2. ドライブの架電ログ／商談文字起こしから、**集計期間**を特定する。
3. その期間が4章に `### 分析済み｜{期間}` として**既にあればスキップ**。
4. 無い期間だけ分析し、4章に追記する（フォーマットは §3）。
5. カルテ冒頭やINDEXのステータスが変わるなら更新。

### 分析済みマーカー仕様
4章の各ログは必ずこの見出しで始める：
```
### 分析済み｜架電｜2026-05-30〜2026-06-17
### 分析済み｜商談｜2026-06-17｜むらさき
```
- `架電` / `商談` の種別を必ず入れる。
- 商談は相手企業名まで入れる（同日複数商談を区別するため）。
- **この見出しが既にあれば、その期間・その商談は再分析しない。**

---

## 3. 出力フォーマット（4章へ追記する形）

### 3-A. 架電分析（架電ログ1期間ごと）
```
### 分析済み｜架電｜{期間}
**数値**：電話{n} / 着電{x}% / 受付拒否{x}% / 対象者通話{x}% / 対象者拒否{x}% / アポ{n}（{x}%）
**ボトルネック**：{受付前 / 受付後 / 本人接続後 のどこで落ちているか}
**本人拒否の分類**：{理由カテゴリと件数。ニーズなし/あいまい断り/導入済/価格不明/タイミング/決裁権なし/自社対応 等}
**所見**：{1〜3行。なぜ落ちているかの仮説}
→ 架電台本原材料：{台本のどの部分をどう直すか}
```

### 3-B. 商談分析（商談文字起こし1本ごと）
```
### 分析済み｜商談｜{日付}｜{相手企業}
**結果**：{アポ / フォロー / Cヨミ / Bヨミ / Aヨミ / 受注 / 失注}
**刺さった一言**：{相手の反応が良かった発話}
**詰まった瞬間**：{相手が引いた・反論した箇所}
**反論と切り返し**：{出た反論／その場の返し／改善案}
→ 商談台本原材料：{台本のどの部分をどう直すか}
```

---

## 4. 勝ちパターンの定義

**「勝ち」＝二次商談トスアップに到達した回**（＝一次商談で温度が上がり、次に繋がったもの）。
- 受注だけを勝ちにしない（営業サイクルが長いため）。
- 二次トスアップ／Bヨミ以上／本人資料請求からの再アポ、を勝ち候補として拾う。
- 各勝ち回について「**何が効いて次に繋がったか**」を1行で記録し、総括の勝ちパターン抽出に回す。

---

## 5. 総括レポートの作り方（全社・全期間）

`_analysis_summary_{YYYYMMDD}.md` として sales-os 直下に出力。**スキップした期間の4章本文も全部読んで材料にする。**

構成：
1. **架電台本修正の原材料**：全社の架電所見を束ねる。共通ボトルネック（例：本人接続後で落ちる現象が複数社に共通）を抽出。
2. **商談台本修正の原材料**：全社の商談所見を束ねる。よく出る反論TOP・刺さった一言の共通項。
3. **勝ちパターン抽出**：二次トスアップに至った回を全社から集め、共通する「効いた要素」を言語化。
4. **横展開メモ**：1社で効いた打ち手を、似た構造の他社へ展開する提案。

---

## 6. エージェントへの起動指示（コピペ用）

```
sales-os/_analysis_protocol.md に従って全社を回せ。
各社カルテ4章の「分析済み」見出しを確認し、未分析の期間・商談だけを分析して4章に追記。
動画本体は無視し、文字起こし・SU定例・SCデータのみ使う。
最後に、スキップ分の4章本文も読んだうえで _analysis_summary_{today}.md を作り、
「架電台本原材料／商談台本原材料／勝ちパターン」の3本立てで総括せよ。
書き言葉は「貴社」を使う。情報が無い項目は捏造せず {未確認}。
```

---

## 7. 注意
- アイドマのドライブ（aidma-hd.jp所有）の閲覧権限が、稼働アカウントに通っている前提。
- 動画mp4は読めない。文字起こしが無い商談は `{文字起こし未取得}` として総括対象外にする。
- 数値はSU定例の表記をそのまま採用し、勝手に再計算しない（食い違う時は両方併記）。
