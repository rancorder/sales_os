## 2. 商材の核・USP / 課題仮説

### 2.1 営業OS上の商材定義

AGERUケアは、単なるリラクゼーションや福利厚生ではなく、以下として定義する。

```txt
人事・総務側では見えにくい従業員の不調・疲労・働きづらさを、
匿名性を保ちながら見える化する取り組み。
```

### 2.2 弱い見せ方

```txt
福利厚生サービスです
ボディケアです
3ヶ月無料です
良ければ導入してください
```

この見せ方では、担当者が「良さそう」と感じても、社内判断・決裁接続まで進みにくい。

### 2.3 強い見せ方

```txt
従業員の本音やコンディションが見えづらい企業様向けに、
まずは無料モニターとして簡易的に状態を可視化し、
人事・総務施策の判断材料になるレポートをお返しする取り組みです。
```

### 2.4 課題仮説

- 人事・総務は、従業員の疲労・不調・働きづらさを事前に把握しにくい。
- メンタル不調・離職・休職が起きてからでは遅い。
- ただし、導入前から大きな費用や運用負荷をかける判断はしにくい。
- そのため、まずは1回無料実施とレポート返却で「自社にも関係がある」と感じてもらう必要がある。

## 3.1 全体フロー｜モニター募集型

```txt
テレアポ
↓

↓
一次相談
  - モニター制度の説明
  - 企業側に返せる価値の説明
  - 1回無料実施の合意
  - 実施に必要な情報の確認
↓
1回無料実施
  - アンケート / 簡易チェック
  - URL / QR / 社内配布
  - 回答期限を切る
↓
二次相談
  - 結果レポート共有
  - 継続して見る意味の説明
  - 3ヶ月無料トライアル提案
  - 決裁者・関与者・社内共有論点の確認
↓
3ヶ月無料トライアル / 2.5次相談 / 社内共有資料 / 中長期フォローへ分岐
```

## 3.2 テレアポ設計

### 目的

テレアポの目的は、3ヶ月無料トライアルの即決ではなく、一次相談の獲得。

### 方針

- 「3ヶ月無料」を前面に出しすぎない。
- 「モニター募集」「従業員コンディション可視化」「レポート返却」を入口にする。
- お願いベースに寄せすぎると「忙しい」「ニーズがない」に弱くなるため、相手が得られる価値を必ず添える。

### 入口キーワード

```txt
モニター募集
従業員コンディションの見える化
匿名アンケート
簡易レポート返却
人事・総務施策の判断材料
1回無料実施
```

```txt

そのため、3ヶ月程度の期間で継続的に見ていくことで、
一時的な不調なのか、
部署や職種ごとに傾向があるのか、
早めにケアすべき兆候があるのか、
より判断しやすくなります。
```

# 9. 判断ログ

## 2026-07-02｜3ヶ月無料は入口訴求ではなく二次相談後の前進条件にする

### 判断

「3ヶ月無料」はテレアポで強く出さず、一次相談後の1回無料実施、二次相談でのレポート共有を経て、3ヶ月無料トライアルに接続する。

### 理由

- テレアポ段階で無料を強く出すと、サービス価値が安く見える。
- 無料でも、全社展開や社内案内が絡むため担当者即決にはなりにくい。
- 相手にとっての参加理由は「無料」ではなく「従業員コンディションの可視化レポートが返ること」。
- AGERUケアは、実績・データ・フィードバックを集めながら育てる商材として扱う方がよい。

### 営業OSへの示唆

```txt
無料オファーは入口の餌ではなく、
相手が社内判断しやすくなる検証導線として使う。
```

# 10. 禁止・注意

- 「福利厚生サービスです」だけで終わらせない。
- 「3ヶ月無料です」をテレアポで前面に出しすぎない。
- モニター募集を単なるお願い営業にしない。
- 二次相談で担当者即決を前提にしない。
- 社内共有先・決裁者・関与者を確認せずに終わらない。
- レポート価値を見せずに3ヶ月無料トライアルへ進めない。
