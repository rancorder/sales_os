## 3. 商談設計

### 3.1 現時点の基本導線

Drive内のテンプレート・コックピット・取材MTGから見る限り、立ち上げ型の営業支援案件として扱う。

```txt
取材MTG
↓
営業資料・送付文面・トーク作成
↓
リスト作成
↓
営業プレイヤー／SCアカウント準備
↓
共有MTG
↓
架電開始
↓
資料送付 / コールバック / アポイント取得
↓
商談実施
↓
営業結果進捗確認
↓
原因・要因・対策確認
↓
定例MTG
```

### 3.2 架電時に必ず残す情報

| 項目 | 残す理由 |
|---|---|
| 接続部署 | 受付突破先が正しいか確認するため |
| 担当者の役割 | 人事・採用責任者に接続できているか見るため |
| 反応した言葉 | トークの勝ちワードを抽出するため |
| 拒否理由 | 商材拒否／時期／予算／既存取引／受付拒否を分けるため |
| 検討関与者 | 商談化後に誰を巻き込むべきか判断するため |
| 資料送付後の次接点 | 送って終わりを防ぐため |

### 3.3 資料送付で止めない設計

この案件で危ないのは、資料送付がゴール化すること。

```txt
悪い流れ：資料送付 → 送付済み → CB取れず → 温度感不明
良い流れ：資料選定ヒアリング → 資料送付 → 専門担当からの連絡時間確認 → 次接点登録
```

ポイントは、資料送付を「逃げ道」ではなく「次回接点の理由」にすること。

### 5.2 初期改善方針

商材確定前のため、いきなりトーク改善に入らない。  
まずは以下の3分解で確認する。

```txt
リストの問題：人事・採用責任者が本当に対象か。業種・企業規模・採用状況は合っているか。
トークの問題：《事業内容》《提案内容》などテンプレ変数が具体化されているか。
運用の問題：資料送付後のCB、SC登録、日報、定例資料への反映が途切れていないか。
```

### 5.3 べー用の見立て

現時点の危険ポイントは、商材・訴求・着地が未確定のまま、テンプレ台本で走ること。

```txt
テンプレ台本で架電
↓
受付突破できない
↓
資料送付で逃げる
↓
CBが取れない
↓
アポが出ない
↓
リストが悪いのか、トークが悪いのか、商材が悪いのか分からなくなる
```

この案件は、最初に「誰の、何の判断を前に進める営業なのか」を1行で定義する必要がある。

```txt
福美人株式会社は、{対象部署}に対して、{解決する課題}を整理し、{商材}の検討接点を作る案件である。
```
