# 追記候補｜発注タイミング特定サービス

## 種別
新規商材/横断知見。必要に応じて `clients/` ではなく `patterns/` または `docs/` に統合する。

## 核となる再定義
「営業リスト構築サービス」ではなく、**製造業特化の発注タイミング特定サービス**として設計する。

## NG訴求
- リスト作成
- 営業先一覧の提供
- 件数勝負

## 正しい訴求
- 開発シグナルから、案件が発生する瞬間を特定する。
- 特許、求人、補助金、設備投資、採用逼迫、高齢化、内製崩壊などをシグナルにする。
- 営業先を探すのではなく、相手が外注を検討する理由が発生したタイミングを探す。

## 導入事例パターン
### 1. 採用逼迫型
- CAD/設計/技術職求人が長期掲載
- 社内リソース不足
- 外部設計/加工/開発支援が刺さる

### 2. 補助金・設備投資型
- 補助金採択
- 設備更新
- 生産性改善投資
- 新設備に伴う周辺設計/導入支援が刺さる

### 3. 高齢化・内製崩壊型
- 代表/技術者の高齢化
- 特定担当者依存
- 図面/ノウハウ/保守が属人化
- 外部支援・標準化・代替リソースが刺さる

## 具体1社ストーリーの必要性
資料には「実際に当てるとこうなる」の1社ストーリーを入れる。

例：
```text
求人120日継続
＋設備更新ニュース
＋代表60代
→ Bランク判定
→ 架電で「ちょうど外注を検討していた」
→ 商談化
```
