## 3. 商談設計

### 3.1 小林フィルター｜営業支援における現場実装OS

小林さんOSの本質は、**商談戦略・資料・フォロー施策を、営業現場で使える粒度に翻訳し、顧客担当者が社内で動ける状態まで整えること**。

別表現にすると、**正しい提案を作るのではなく、顧客・営業担当・社内決裁者の間で情報が詰まらず流れるように、資料・言葉・役割・次アクションを整えるOS**。

この案件では、べー側で作る「商談構造・勝ち筋・仮説」に対し、小林フィルターで「現場で破綻しない運用に落ちるか」を検証する。

---

### 3.2 判定思想：戦略の正しさより、商談で自然に使えるか

小林フィルターでは、戦略を「正しいか」だけで判定しない。
最初に見るのは、**営業担当が商談の中で自然に言えるか／顧客が自分ごと化できるか**。

修正対象になる例：
- 言葉が強すぎる
- 説明が長い
- 顧客が受け取りにくい
- サービス説明に入りすぎて売り込み感が出る
- 正論ワードが営業電話で聞き流される

AGERUケアでは、以下のような強い言葉をそのまま出しすぎない。
- 離職
- 生産性ロス
- 採用教育コスト
- 健康経営
- プレゼンティーズム

置き換えの方向性：
> 最近、人が辞める・休む・本調子で働けない状態が、会社側の見えない負担になっていないかを整理する企業様が増えています。

---

### 3.3 資料思想：何を伝えるかではなく、どの局面で使うか

資料は「増やす」のではなく、**役割を分ける**。
作成前に、必ず用途を切る。

| 資料種別 | 役割 |
|---|---|
| For You提案書 | その企業向けに刺す資料 |
| 業界導入イメージ | 汎用で使い回せる資料 |
| 役員向け判断整理 | 担当者が社内共有しやすい資料 |
| プラン提案 | 導入条件を具体化する資料 |
| 商談進捗レポート | 現状認識と次の打ち手を揃える資料 |

資料作成前の問い：
1. 顧客の理解を進める資料か
2. 担当者の社内共有を助ける資料か
3. 決裁者判断を促す資料か
4. 次回商談の論点を揃える資料か

---

### 3.4 商談フロー思想：標準と例外対応を分ける

ケースバイケースは必要。
ただし、最初から個別対応に逃げない。

まず基本フローを作り、そのうえで温度感・決裁状況・同席状況に応じて例外対応を足す。

| 区分 | 内容 |
|---|---|
| 標準フロー | 1次商談 → 2次商談 → 決裁者接続 |
| 例外対応 | 2.5次商談／電話フォロー／南さん同席／個別資料送付 |
| 確認論点 | 誰が何を担うか、次回以降も再現する運用か |

施策を出すときの問い：
1. これは標準フローか
2. 例外対応か
3. 誰の責任範囲か
4. 次回以降も再現する運用か

---

### 3.5 トーン管理：社内分析用と顧客提示用を混ぜない

商談分析としては必要でも、顧客提示用には出さない方がいい表現がある。

顧客提示用に出さない表現：
- 囲い込み
- 決裁に向けて詰める
- 先方の温度感
- 営業評価
- 商談者の弱点
- 顧客提示用、という内輪表現

運用ルール：
| 用途 | 入れる内容 |
|---|---|
| 社内用 | 本音、仮説、課題、営業評価、温度感、次の打ち手 |
| 顧客用 | 現状整理、確認論点、判断材料、次の進め方 |

---

### 3.6 ゴール判定：担当者が社内で動ける武器を持ったか

商談相手が担当者の場合、その人が納得するだけでは足りない。
重要なのは、**担当者が社内で説明できる状態を作ること**。

商談後の判定基準：
> 相手が「良い」と思ったかではなく、相手が社内で説明できる武器を持ったか。

資料・フォローは以下に寄せる。
- 役員向けに1〜2枚で整理する
- 担当者がそのまま転送できる表現にする
- 判断論点が分かる構成にする
- 導入可否ではなく、検討条件が整理できる内容にする

---

### 3.7 サービス説明の出し方：早く出しすぎない

AGERUケアは、最初から「ボディケア」「福利厚生」と見せるとズレる。
一方で、「離職対策」「生産性改善」「健康経営」と強く言いすぎても営業臭くなる。

基本方針：
1. 先に、顧客が「うちにもありそう」と思える課題の置き場を作る
2. 課題をやわらかく言語化する
3. 後半でAGERUケアに接続する
4. サービス説明より、検討条件・社内説明材料を整える

### 3.8 前進設計：きれいな正解より、次に進む確率を見る

理想ルートだけを追わない。
商談を止めないための中間アクションを設計する。

| 理想 | 現実解 |
|---|---|
| 決裁者同席 | 難しければ2.5次商談 |
| 2次商談で決裁接続 | 重ければ電話フォロー |

| 温度感が低い | 南さん同席などで受け止め役を足す |

判断基準：
> 理想ルートで進まない時、商談を失注させずに温度を保つ中間アクションがあるか。

### 3.9 モニター募集型｜一次・二次相談シナリオ骨組み

#### 全体フロー

```txt
テレアポ
↓
モニター募集として一次相談を獲得
↓
一次相談
  - モニター制度の説明
  - 企業側に返せる価値の説明
  - 1回無料実施の合意
  - 実施に必要な情報の確認
↓
1回無料実施
  - アンケート / 簡易チェック
  - URL / QR / 社内配布
  - 回答期限を切る
↓
二次相談
  - 結果レポート共有
  - 継続して見る意味の説明
  - 3ヶ月無料トライアル提案
  - 決裁者・関与者・社内共有論点の確認
↓
3ヶ月無料トライアル / 2.5次相談 / 社内共有資料 / 中長期フォローへ分岐
```

#### 一次相談の目的

```txt
1. AGERUケアのモニター制度を理解してもらう
2. 1回無料実施への不安をなくす
3. 実施に必要な情報を確認する
4. 1回無料実施の合意を取る
```

一次相談のゴールは、契約ではなく「1回無料実施の合意」。

#### 一次相談の流れ

```txt
1. 冒頭あいさつ・本日の目的共有
2. モニター募集の背景説明
3. 企業側に返せる価値の説明
4. AGERUケアの概要説明
5. 1回無料実施の流れ説明
6. 実施負荷の軽さを説明
7. 実施に必要な確認項目のヒアリング
8. 実施期限・二次相談日程の設定
9. クロージング
```

#### 一次相談で確認する項目

```txt
・対象人数
・対象部署
・回答対象者の属性
・社内共有方法
  - メール
  - LINE
  - 社内チャット
  - 紙 / QRコード
・回答期限
・従業員に案内する際の文面の有無
・人事側で追加確認したい項目の有無
・レポート共有先
・二次相談に参加すべき関係者
```

#### 二次相談の目的

二次相談の目的は、1回無料実施の結果を共有し、企業側に「継続して見る価値がある」と感じてもらうこと。

表向きのゴール：
```txt
3ヶ月無料トライアルへの合意
```

裏ゴール：
```txt
・社内共有先
・決裁者 / 関与者
・導入判断で止まりそうな論点
・次回、誰を巻き込むべきか
```

#### 二次相談の流れ

```txt
1. 冒頭あいさつ・実施のお礼
2. 本日の目的共有
3. 実施結果レポートの共有
4. 結果から見える示唆の説明
5. 継続して見る意味の説明
6. 3ヶ月無料トライアルの提案
7. 導入判断に必要な論点確認
8. 決裁者・関与者・社内共有先の確認
9. 次回アクションの合意
```

#### 二次相談クロージング分岐

| 分岐 | 次アクション |
|---|---|
| A. 3ヶ月無料トライアル合意 | 対象者、開始時期、案内方法、回答期限、レポート共有方法を確定する |
| B. 社内確認 | 社内共有先、判断者、懸念論点を確認し、必要に応じて社内共有資料を作成する |
| C. 温度感はあるが判断者不在 | 2.5次相談または3次相談を設定し、決裁者または関与者の同席を依頼する |
| D. 今すぐ難しい | 時期、予算、運用負荷、対象範囲のどこで止まっているかを整理し、中長期フォローへ回す |

### 2026-07-02｜3ヶ月無料は入口訴求ではなく二次相談後の前進条件にする

判断：
「3ヶ月無料」はテレアポで強く出さず、一次相談後の1回無料実施、二次相談でのレポート共有を経て、3ヶ月無料トライアルに接続する。

理由：
- テレアポ段階で無料を強く出すと、サービス価値が安く見える。
- 無料でも、全社展開や社内案内が絡むため担当者即決にはなりにくい。
- 相手にとっての参加理由は「無料」ではなく「従業員コンディションの可視化レポートが返ること」。
- AGERUケアは、実績・データ・フィードバックを集めながら育てる商材として扱う方がよい。
