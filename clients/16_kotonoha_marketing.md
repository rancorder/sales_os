## 2. 商材の核 / USP

### 2.1 商材①：RBL企業協賛

RBLは、2026年開幕予定の3x3プロバスケットボールリーグ。  
3x3は東京オリンピック正式種目であり、音楽・ファッション・ストリートカルチャーと相性が高いアーバンスポーツとして打ち出されている。

営業上の核は、単なるスポーツ協賛ではない。

```txt
広告枠を買う
ではなく、
若者カルチャーに企業が参加する。
```

RBL協賛の提供価値：

- 若年層への認知拡大
- 採用ブランディング
- イベントプロモーション
- SNS露出
- 都市型イベントでのブランド体験
- スポーツ・音楽・ファッションを横断した接点づくり

リーグ訴求で使える主要情報：

| 項目 | 内容 |
|---|---|
| 開幕 | 2026年予定 |
| 年間来場見込み | 15万人以上 |
| メインターゲット | 10〜30代 |
| 開催予定都市 | 京都・東京・名古屋・福岡・博多・山梨 |
| 提供メニュー | 会場内広告掲出、ユニフォームロゴ掲載、イベントブース出展、SNS/配信露出、コラボキャンペーン |
| ポジショニング | 従来型スポンサーではなく、若年層接点・採用広報・体験型マーケティング |

### 2.2 商材②：AI検索対策（AIO対策）

AIO対策は、従来のSEOだけに頼らず、ChatGPT等のAI検索時代において**AIに自社・サービスが適切に言及される状態**を作る支援。

営業上の核は、検索順位ではなく、AI回答内での想起・言及である。

```txt
検索結果で上に出る
ではなく、
AIが答える時に候補として扱われる。
```

AIO対策の提供価値：

- AIが参照しやすい情報構造の設計
- 外部発信を通じた信頼できる情報源化
- SEO依存からの脱却
- Google検索外への露出導線
- 競合が少ない段階での先行ポジション獲得
- 認知・指名検索につながる情報資産の蓄積

提供形態：

- 業務委託での伴走支援
- 情報整理 / コンテンツ設計 / 外部発信の仕組み化
- 社内リソースを過度に増やさず、実務をカバーする形で推進

## 3. 商談設計OS

### 3.1 最重要判断：RBLとAIOを混ぜない

この案件は、同じクライアントでも商材の買われ方がまったく違う。

| 商材 | 相手の主な関心 | 判断者 | 商談の勝ち筋 | 失速要因 |
|---|---|---|---|---|
| RBL協賛 | 若年層認知、採用広報、地域/イベント露出、ブランド体験 | 広報、マーケ、採用、人事、経営 | 協賛を広告ではなく「若者カルチャー参加」に変換 | 協賛=大企業向け、効果測定が曖昧、予算枠なし |
| AIO対策 | Web集客、SEO停滞、AI検索時代への不安、情報発信体制 | 経営、マーケ、Web担当、広報 | SEOではなく「AIに選ばれる情報設計」に変換 | AIOの概念が新しく、緊急度が低く見える |

営業OS上の扱い：

```txt
RBL = 協賛・採用広報・若年接点OS
AIO = AI検索・情報設計・Web集客OS
```

同じリスト、同じトーク、同じ商談資料で回すと、刺さった理由と拒否理由が混ざり、改善不能になる。

### 3.2 RBL一次商談の型

RBLの一次商談では、最初から協賛メニューを説明しすぎない。

推奨順序：

```txt
若者は御社を知っているか
↓
採用したい人材に届いているか
↓
採用ブランディング / 若年層認知の現状確認
↓
従来のスポーツ協賛イメージを確認
↓
RBLは広告枠ではなく、都市型・体験型・SNS拡散型の接点であると再定義
↓
具体的な協賛プラン・費用・露出・事例を次回で提示
```

一次商談の着地は、即決ではなく二次トスアップ。

二次で持ち込むべき論点：

- 具体的な協賛プラン
- 費用感
- 露出内容
- 過去事例 / プレ大会情報
- どの目的で協賛するか（採用・認知・地域・SNS・ブース）
- 社内決裁者 / 相談先

### 3.3 AIO一次商談の型

AIOは概念が新しいため、いきなり「AIO対策です」と言うより、相手の検索行動変化に接続する。

推奨順序：

```txt
Web集客 / SEO / 広告の現状確認
↓
生成AIやChatGPTを業務で使っているか確認
↓
顧客がAIに聞いて比較する時代の入口を提示
↓
SEOは検索結果、AIOはAI回答内の言及だと切り分ける
↓
自社情報がAIに拾われやすい構造になっているかを論点化
↓
次回で現状診断・情報構造・発信設計の提案へ進める
```

二次で持ち込むべき論点：

- 現状のWeb集客施策
- SEO / 広告 / SNS / コンテンツ運用状況
- 生成AI利用状況
- 情報発信リソース
- AIOで狙うテーマ・検索文脈
- 支援範囲と社内工数

## 5. 営業判断 / 課題

### 5.1 定量から見えること

対象者通話率は5〜6%台で推移しており、担当者接続までの母数は一定作れている。  
一方で、対象者拒否率が72〜88%と高く、担当者に接続した後の初期関心形成で落ちている可能性が高い。

つまり、単なる受付突破だけでなく、本人接続後の一言目・論点設定が重要。

```txt
受付突破の問題
だけではなく、
本人接続後に「自分ごと化」できていない問題。
```

### 5.2 乖離の見立て

原因：

- RBL協賛とAIO対策で、刺さる部署・課題・判断理由が異なる
- RBLは「協賛」「スポンサー」という言葉だけだと、広告予算・大企業向け・効果不明に聞こえやすい
- AIOは「AI検索対策」という概念が新しく、緊急度が伝わらないと情報収集で止まりやすい
- 資料請求は発生しているが、商談化率が高くないため、資料送付後の再接続設計が弱い可能性

改善：

- 商材別にリスト・トーク・資料・KPIを分ける
- RBLは「協賛」ではなく「若年層接点・採用広報・体験型マーケティング」の入口で話す
- AIOは「AIに自社がどう出るか」という診断型の入口で話す
- 資料請求先を放置せず、資料送付後の確認トークを設計する

対策：

- RBL用のターゲット分類：採用強化 / BtoC認知 / 地域露出 / 若年層接点 / イベント活用
- AIO用のターゲット分類：SEO停滞 / Web問い合わせ依存 / 専門性が伝わりにくい / 競合比較されやすい / 代表・広報主導
- 対象者拒否ログを「拒否理由」ではなく「どの言葉で切られたか」に分解する
- 資料請求先は、送付後に「どの観点なら社内共有できそうか」を確認する

```txt
架電量は十分に積み上がっているが、アポ率は低下傾向。
受付突破よりも、本人接続後の関心形成に課題がある。
```

定性的な報告案：

```txt
今月は電話数831件に対してアポイント2件、本人資料請求4件となりました。

RBL協賛については「スポンサー募集」として伝えると広告予算・大企業向けの印象が強くなるため、若年層認知・採用ブランディング・都市型イベント接点として再定義して伝える必要があります。

AIO対策については、AI検索対策という新規性はあるものの、緊急度が伝わらないと情報収集で止まりやすいため、「自社がAIにどう認識されているか」という診断型の入口に変換する方針です。
```

### 7.2 次回改善アクション

1. RBL / AIOで架電ログを分けて再集計する
2. 対象者拒否ログを、拒否理由ではなく「切られた表現」で分類する
3. RBLは採用・若年層認知・地域露出に刺さる企業を優先する
4. AIOはSEO停滞・Web集客依存・専門性訴求が必要な企業を優先する
5. 資料請求先へ、資料到着確認ではなく「どの観点なら社内共有できそうか」を聞く
6. 次回商談では、RBLは協賛プラン・費用・露出、AIOは現状診断・情報構造・支援範囲を提示する
