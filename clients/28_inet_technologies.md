## 2. 商材の核 / USP

株式会社アイネットテクノロジーズ案件は、M365利用企業・導入検討企業に対して、単なるライセンス提案ではなく、以下を整理する案件として扱う。

```txt
Microsoft 365 / Office 365 を安全に使える状態にする
↓
導入・運用・セキュリティ・監視・SOC・社内展開の論点を整理する
↓
情シスだけでなく、企画・経営企画・DX推進・セキュリティ責任者へ接続する
↓
必要に応じてSCS評価制度・サプライチェーン要件の文脈で経営課題へ引き上げる
```

### 2.1 有効な入口

- Microsoft 365 / Office 365 を現在利用しているか
- これからMicrosoft 365を導入・全社展開しようとしているか
- 新しく入れるサービスのセキュリティが問題ないか確認したいか
- EDR / EPP / SOC / FW / VPN / DR / 資産管理 / AD / MDM などの運用が整理されているか
- SCS評価制度やサプライチェーン上のセキュリティ要件に備える必要があるか

### 2.2 刺さる部署仮説

| 部署・役割 | 刺さる論点 |
|---|---|
| 情報システム / セキュリティ推進 | M365運用、端末管理、EDR/EPP、SOC、資産管理、脆弱性対策 |
| 経営企画 / 企画部 | 全社展開、セキュリティ投資、サプライチェーン要件、社内統制 |
| DX推進 / 業務改革 | M365導入、GoogleからMicrosoftへの移行、業務基盤刷新 |
| 代表 / 事業責任者 | セキュリティ担当を雇うより安い、外部監視・運用の仕組み化、取引条件対策 |
| MS / パートナー連携先 | Microsoft関連リード・既存顧客・SMB顧客との接続 |

## 3. 商談設計

### 3.1 基本導線

```txt
過去資料請求・セミナー・体験会・Webinar・M365導入情報
↓
フォローコールで現状確認
↓
M365利用状況・導入予定・検討部署を確認
↓
セキュリティ課題を直接断定せず、運用・展開・取引要件として整理
↓
必要部署・関与者を特定
↓
商談化 / 先方側対応 / SU商談 / 代表同席 / MS連携のいずれかに振り分け
```

### 3.2 リード判定

| 判定 | 条件 | 対応 |
|---|---|---|
| S | M365導入予定、全社展開予定、予算時期あり、検討中心人物と接続済 | 即フォロー。先方側営業またはSU上位担当へ接続。 |
| A | M365利用中、セキュリティ・情シス・経営企画の接点あり | SUフォローコール優先。課題と関与部署を確認。 |
| B | 過去資料請求、Webinar、セミナー、体験会、本人資料送付 | フォローコールで温度感判定。資料送付後の再接触を設計。 |
| C | Google中心、M365利用不明、対象部署不明、情報不足 | 精査後に架電。無理に商談化しない。 |
| D | 地方自治体など今回対象外、明確に商材対象外 | 除外・禁止・リストから削除。 |

### 3.3 商談内で確認すること

- Microsoft 365 / Office 365 の利用状況
- 利用ライセンス、利用人数、主要利用製品
- AD / MDM / 資産管理 / EDR / SOC / 脆弱性対策の状況
- 新規導入・全社展開・移行予定の有無
- 実際の検討部署と関与者
- 予算確定時期
- セキュリティ上の困りごとではなく、運用上・取引上・社内展開上の懸念
- 次回商談に誰を巻き込むべきか

### 3.4 禁止する進め方

- 「セキュリティで困ってますか」だけで終わる
- 情シス担当者だけを決裁者扱いする
- M365利用有無を聞いただけで温度感を判断する
- Google利用企業に同じ訴求を押し込む
- 資料請求リードを一律で同じ温度感として扱う
- 商談後登録・サンクスメール・リマインドを個人の記憶で管理する

## 4. 進捗ログ

### 4.1 membersチャット抽出からの要点

- 過去資料請求、PIIフォロー、Webinarフォロー、M365 / Office365導入企業リストを中心に稼働。
- SPでアポが出ない期間があり、SU担当者をフォローコール・商談同席・商談主導へ育成する方針へ移行。
- 過去資料請求先にSUを当てたところ、受付拒否0、対象者通話率が高く、複数アポが発生。
- M365をこれから導入する企業、全社展開予定企業、新サービスのセキュリティを気にしている企業が高優先リードとして浮上。
- セキュリティ担当から、実際の検討部署が経営企画・企画部側にあると判明するケースがあり、部署接続の重要性が確認された。
- Outlook / Teams / SalesCrowd / Mazrica / クライアントメールを跨ぐ運用が複雑化し、サンクス・リマインド・登録漏れが発生。
- SU稼働起因で解約要望を含むアラートが発生し、上長説明・改善提案が必要な状態になった。
- 録画・音声取得の不備が複数回発生し、商談分析に支障が出た。
