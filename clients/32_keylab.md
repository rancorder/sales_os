## 2. 商材の核 / USP

### 2.1 一言定義

```txt
POINTQUICは、LINEを入口に会員証・ポイント・顧客管理・販促配信・来店履歴をつなぐ店舗CRM基盤である。
```

単なるポイントカードではなく、店舗側の本質価値は以下。

```txt
・会員証発行の摩擦を下げる
・紙カード/スタンプカードの紛失・管理負担を減らす
・ポイント利用履歴、来店頻度、会員情報を一元管理する
・LINE経由で販促メッセージやクーポンを打てる
・POS/EC連携、Web会員証、NFC/ICタッチなどに拡張できる
```

### 2.2 営業上の強み

```txt
1. 顧客側の登録摩擦が小さい
   LINE友だち追加で会員化し、カード発行やメール登録の手間を減らせる。

2. 店舗側の運用が軽い
   iPhone/iPadと無料アプリをポイント端末として使えるため、専用端末前提ではない。

3. 顧客管理と販促がつながる
   会員情報、ポイント残高、利用履歴、来店頻度を見ながら、条件抽出・LINE配信につなげられる。

4. 既存運用から段階的に拡張できる
   カード併用、LINE会員証、店舗アプリ、NFC/ICタッチ、POS/EC連携など、現状に合わせた提案余地がある。

5. 導入実績の幅が広い
   公式サイト・営業資料上では、飲食、商業施設、ホテル、イベント/来場管理、大手企業など複数用途の事例が確認できる。
```

### 2.3 注意すべき表現

```txt
NG：ポイントカードを新しく作りませんか？
OK：今の会員管理やリピート施策を、LINE・Web会員証・ポイント履歴まで含めて整理しませんか？

NG：LINE販促なら売上が上がります
OK：会員証・来店履歴・配信導線が分かれると施策が打ちづらくなるため、まず運用を一元化する提案です

NG：サブスク移行が必須です（架電冒頭で強く言う）
OK：今後のサポート・新機能・安定運用に関わるため、現在のご利用状況に合わせて個別にご案内しています

NG：新しいサービスの営業でお電話しました
OK：現在ご利用中のPOINTQUICの継続利用・設定確認に関する重要なご連絡です

NG：登録しないと止めます
OK：システム仕様上、登録がない場合に自動でロック/停止される可能性があるため、継続利用条件をご説明します
```

## 3. 営業設計

### 3.1 直近の主戦場

```txt
対象：アップセル訴求で取得した資料請求先 約37〜40件
目的：POINTQUIC2.0の設定方法・新機能・拡張機能に関するZoom個別説明アポ取得
着地：予約フォーム入力 → 鍵野代表とのZoom面談
```

この架電は、完全新規の課題喚起ではなく「すでに資料請求/関心表明した相手への温度維持」である。  
そのため、冒頭からサービス説明をやり直すより、以下の順番がよい。

```txt
1. 資料請求 / ご案内済み文面の確認
2. POINTQUIC2.0の設定確認・新機能説明枠であることを伝える
3. 相手の関心が、初期設定・店舗操作・LINE連携・POS/EC連携・会員証運用のどれかを確認する
4. 代表面談/Zoomで確認した方が早い論点として予約フォームへ接続する
5. 取れた関心タグをログに残す
```

### 3.2 フェーズ分離

```txt
SP：資料請求先への架電、個別説明枠の案内、予約フォーム誘導、関心タグ取得
IS：相手の現行運用・関心機能・検討時期・関与者を深掘りする差分検証
SU/代表面談：現行運用の整理、移行/拡張の判断条件確認、設定・新機能説明、次アクション設計
商談代行化：代表商談に同席・録画し、説明台本/FAQ/反論処理へ変換する
SM/PM：架電ログ分析、拒否理由分類、台本改善、定例資料反映
```

### 3.3 SP架電で必ず拾う情報

```txt
・現在POINTQUICをどの範囲で利用しているか
・POINTQUIC2.0で確認したいことは何か
・初期設定 / 店舗操作 / 会員証表示 / ポイント処理 / LINE連携 / EC連携 / POS連携 のどれに関心があるか
・実際に操作する担当者と、契約/支払い判断をする人は同じか
・無料利用終了やサブスク移行に関する認識があるか
・Zoom説明を受ける場合、誰が参加した方がよいか
・今すぐ設定したいのか、今後の活用相談なのか
・すでに登録済み/資料請求済み/説明予約済みではないか
・クレーム注意/継続意思なし/担当不明など、再架電時に事故る情報はないか
```

### 3.8 リスク管理

```txt
1. 営業電話扱いされるリスク
   「新サービス案内」ではなく「既存利用システムの重要連絡」として入る。

2. クレーム化リスク
   無料契約認識、告知不足、カード費用、停止/ロック表現に注意する。
   強く押すより、継続利用条件の確認・説明に寄せる。

3. 登録済み顧客への重複架電リスク
   サブスク登録済み、説明予約済み、対象外、継続意思なしをリストへ即時反映する。

4. 業態別時間帯リスク
   美容室、バー、教室系などは通常法人時間だけでは接続しづらい。
   業態別に架電時間を調整する。
```

架電時間の考え方：

```txt
整体・接骨院：昼休み/夕方前後
美容室：午前遅め or 夕方
バー：15時以降
スクール：レッスン前後
飲食：アイドルタイム
```
