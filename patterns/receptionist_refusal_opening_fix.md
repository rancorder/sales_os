# 受付拒否 冒頭改善パターン

## 目的
受付で営業判定される前に、担当者へつながる確率を上げる。

## 原則
- 冒頭で商材説明をしない。
- 「サービス」「ご提案」「営業」「資料送付」を早い段階で出さない。
- 受付には判断させず、担当確認に徹する。

## NG例
```text
弊社は〇〇サービスを提供しておりまして、担当者様にご提案でお電話しました。
```

## OK例
```text
〇〇の件で、担当の方に1点だけ確認したくお電話しました。
```

## パターン
| リスト | 冒頭フック |
|---|---|
| 展示会出展企業 | 展示会での説明資料・映像活用の件で、担当の方に1点確認です |
| 採用課題企業 | 求人票や採用ページの見せ方の件で、採用ご担当に1点確認です |
| 製造業設計 | 設計業務の外部対応可否の件で、技術/設計ご担当に確認です |
| 自治体 | 避難所や施設運用の確認で、防災/施設担当の方に確認です |
| 幼稚園 | 園内で実施できる創造系プログラムの件で、園長先生か担当の方に確認です |

## KPI
受付突破率を一次指標にする。アポ率は二次指標。
