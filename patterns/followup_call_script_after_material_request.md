# 追撃架電パターン｜本人資料請求フォロー

## 0. 目的

本人資料請求で止まった先を、資料送付後のフォロー架電で再接触し、二次商談または次回接点へつなげるための汎用パターン。

このパターンは、株式会社ジェネレートワンの本人資料請求ログ分析から抽出したが、他案件にも横展開できる。

## 1. 基本原則

本人資料請求者は、新規架電より温度は高いが、高温リードとは限らない。

資料請求には、以下の複数の意味がある。

| タイプ | 本音 | 追撃の勝ち筋 |
|---|---|---|
| ① 資料を見てから判断が社内ルール | 電話口だけでは判断できない | 資料の見る論点を絞らせる |
| ② 担当者・部署が別にいる | 本人に決裁権や担当権限がない | 本人を紹介者に変える |
| ③ 予算・時期待ち | 今は決められないが、後で検討余地がある | 予算が決まる前の判断材料にする |
| ④ 営業電話が多く警戒 | まず資料で見極めたい | 押さずに1点だけ確認する |
| ⑤ 断り代替 | 会話を終えるために資料請求している | 深追いせず、次回時期か用途だけ取る |

## 2. NGパターン

本人資料請求フォローでやってはいけないこと。

- 「資料をご覧いただけましたか？」だけで入る
- 見ていない相手に資料説明を最初からやり直す
- 資料請求者を一律に高温リード扱いする
- 相手が資料を求めているのに、同じ商談打診を繰り返す
- 相手の関心軸を確認せずに商材説明する
- 社内共有先・担当部署・次回確認時期を取らずに終話する
- 「興味があればご連絡ください」で主導権を手放す

## 3. 冒頭トーク

```txt
お世話になっております。
先日、資料をお送りした件でご連絡しました、〇〇の〇〇です。

資料を細かくご説明したいというより、
御社の場合、どの用途で見るのが近そうかだけ確認できればと思いまして。

今、1〜2分だけよろしいでしょうか。
```

## 4. 資料を見ていない場合

```txt
全然大丈夫です。
今回の資料は一般的な内容なので、
むしろ御社で見るならどの観点が近いかを先に整理した方が、
社内共有もしやすいと思っています。

採用・広報・営業・業務改善の中だと、
どれが一番近そうでしょうか。
```

※商材に応じて「採用・広報・営業・業務改善」の部分は差し替える。

## 5. タイプ別の本線

### ① 資料を見てから判断が社内ルール

- 相手は資料を通過儀礼として扱っている
- こちらが見る論点を絞る

```txt
ありがとうございます。
資料は一般向けなので、
御社で見るならどの観点が近いかを絞った方が判断しやすいと思います。

〇〇向け・〇〇向け・〇〇向けの中だと、
どれが一番近そうでしょうか。
```

クロージング：

```txt
その観点であれば、次回は一般説明ではなく、
御社向けの簡単な活用イメージをお持ちできます。
15〜20分だけ確認の場をいただけませんでしょうか。
```

### ② 担当者・部署が別にいる

- 本人を口説かない
- 本人を紹介者に変える

```txt
ご担当が別部署とのこと、承知しました。
差し支えなければ、その方にお渡しいただく前提で、
どの観点で見ていただくのが近そうかだけ確認させてください。
```

橋渡し依頼：

```txt
可能でしたら、資料をお渡しいただく際に、
「この観点で見ると判断しやすい」と一言添えていただけると助かります。

社内共有後、〇日頃にご感触だけ確認してもよろしいでしょうか。
```

### ③ 予算・時期待ち

- 「今は時期ではない」を否定しない
- 予算が決まる前に入る

```txt
承知しました。
今すぐ決める前提でなくて大丈夫です。

むしろ予算や来期施策を決める前に、
判断材料として御社向けの活用イメージだけ見ておく形でも大丈夫です。
```

クロージング：

```txt
決める場ではなく、判断材料を整理する場として、
15〜20分だけお時間いただけませんでしょうか。
```

### ④ 営業電話が多く警戒

- 押さない
- 短く切り上げる姿勢を見せる

```txt
承知しました。
不要であればもちろん大丈夫です。

最後に一点だけ、
御社で見るとしたら、〇〇・〇〇・〇〇の中だとどれが一番近そうでしょうか。
```

回答がある場合：

```txt
ありがとうございます。
では、その観点に絞って資料を見ると判断しやすいと思います。
〇日頃に一度だけ、その後のご感触を伺ってもよろしいですか。
```

回答がない場合：

```txt
承知しました。
では、今回はいったん資料だけご確認いただければ大丈夫です。
不要でしたらもちろん結構ですので、また時期が合いそうな際にご案内します。
```

### ⑤ 断り代替

- 深追いしない
- 次回接触の許可だけ取る

```txt
承知しました。
今回はいったん資料だけで大丈夫です。

もし今後検討されるとしたら、〇月頃など時期の目安はありますでしょうか。
その頃に一度だけ情報提供してもよろしいでしょうか。
```

## 6. 共通切り返し

| 相手の言葉 | NG | OK |
|---|---|---|
| また資料送って | 同じ資料を再送して終わる | 「今回は御社向けに見る観点を1点添えて送ります」 |
| 担当に聞いてみる | 押し切ろうとする | 「社内共有後、〇日頃にご感触だけ伺ってもよろしいですか」 |
| 今は必要ない | 今すぐでなくてもと粘る | 「では〇月頃に一度だけ情報提供してもよろしいですか」 |
| 興味があれば連絡する | そのまま引く | 「こちらからも〇日頃に一度だけ状況確認してよろしいですか」 |
| 資料を見てから | 商談打診を繰り返す | 「見る観点だけ先に整理してもよろしいですか」 |

## 7. 商材別の置き換え例

### ジェネレートワン型

- 誤認：YouTube広告、動画制作
- 再定義：番組出演を起点にした採用・営業・広報資産
- 関心軸：採用、広報/PR、営業前の信頼形成、経営者/事業責任者の見せ方
- 二次商談名：御社向け企画骨子案の確認

### DX/業務改善型

- 誤認：システム販売、ツール導入
- 再定義：現場業務の詰まりを見える化し、改善余地を整理する支援
- 関心軸：工数削減、属人化解消、教育、品質安定
- 二次商談名：現状業務の課題整理・改善仮説の確認

### 採用支援型

- 誤認：求人広告、媒体営業
- 再定義：応募者の質・志望度・ミスマッチ改善の支援
- 関心軸：応募数、応募者の質、採用単価、定着率
- 二次商談名：採用課題別の改善仮説確認

## 8. 追撃KPI

- 資料請求先への再架電到達率
- 再接触で「次回連絡の許可／時期」を取れた率
- 関心軸を特定できた率
- 社内共有先・担当部署を確認できた率
- タイプ別の二次商談化率
- 二次トスアップ数

## 9. 運用メモ

- 本人資料請求リストは、新規架電より優先度が高い
- ただし、高温リード扱いしすぎると失注化する
- フォロー前に、ログからタイプ判定してから架電する
- 効いた切り返しは本パターンに追記する
- 個人名・電話番号・メールアドレス・架電ログ原文はGitHubに置かない
