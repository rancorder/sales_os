# リスト設計：業種×規模×痛みシグナル

## 背景
単なる業種×規模のリストでは、課題未確定アポが量産される。営業支援OSでは、リストを「痛みが発生している可能性の高い順」に並べる。

## 3軸設計
```text
業種 × 規模 × 痛みシグナル
```

## 痛みシグナル例
| 領域 | シグナル |
|---|---|
| 採用 | 求人長期掲載、現場職/専門職不足、採用ページが弱い、写真が少ない |
| 製造業設計 | CAD求人、設計者不足、設備投資、外注先募集、2D→3D化 |
| データ入力 | 紙帳票、繁忙期処理、手入力、BPO求人、対人サービス業 |
| 映像 | 展示会出展、採用強化、難解商材、研修コンテンツ、サイト説明が複雑 |
| 建設 | 現場管理、調達課、積算、EPC、施工管理の求人増 |
| 外国人材 | 季節繁忙、社宅あり、インバウンド対応、外国語対応 |

## S/A/B/D判定
| 判定 | 意味 | 対応 |
|---|---|---|
| S | 痛みシグナルが複数あり、商材親和性が高い | 詳細仮説を作り優先架電 |
| A | 商材親和性は高いが痛み未確定 | 仮説を置いて架電 |
| B | 条件付き適合 | 条件確認後に判断 |
| D | 不適合/競合/予算ズレ | 除外 |

## 使い方
1. HP/求人/ニュース/補助金/展示会情報を見る
2. 痛みシグナルを1社ごとにメモ
3. S/A/B/Dを付ける
4. 架電時に仮説を検証
5. 実戦反応でSランク昇格/降格する
