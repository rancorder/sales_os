# 共通OS｜BPO架電品質管理OS

## 1. 一言定義
BPO・入力代行・AI-OCR・RPA・事務代行・DX支援案件では、アポ数だけで評価しない。

重要なのは、商談前に「具体業務が特定できているか」である。

```txt
弱い成果指標：話を聞いてくれるアポ数
強い成果指標：崩れている具体業務が特定済みの有効アポ数
```

## 2. 背景
BPO系商材は、以下のような抽象訴求に寄りやすい。

- DX化できます
- 工数削減できます
- 入力業務を効率化できます
- 業務負担を減らせます

しかし、この表現では相手に営業電話として処理されやすい。
相手は「困っていない」「すでにシステムがある」「今は大丈夫」「作業量が少ない」で閉じる。

## 3. 基本思想
BPO架電では、サービス説明より先に業務特定を行う。

```txt
サービス説明
↓
興味確認
↓
アポ打診
```

ではなく、

```txt
残っている業務確認
↓
処理者・量・時期の確認
↓
既存システムで拾えていない範囲の確認
↓
外出し余地の確認
↓
有効アポ化
```

とする。

## 4. アポ定義

| 区分 | 定義 | 評価 |
|---|---|---|
| 通常アポ | 話を聞いてくれる・情報提供を受ける | 低確度混入の可能性あり |
| 有効アポ | 崩れている具体業務が特定済み | 商談化優先 |

## 5. 有効アポ条件
以下のいずれか、できれば複数が確認できた場合に有効アポとして扱う。

- 紙・FAX・PDF・申込書・発注書・伝票などの入力物が確認できている
- 入力量・繁忙期・月末処理などの負荷が確認できている
- 現在の処理者が確認できている
- 既存システムで拾いきれていない入力業務が確認できている
- 外部委託・外出しの余地が確認できている

## 6. リスト判定

### Sランク
- 紙/FAX/申込書/伝票/検査記録/レセプト/入力系求人/繁忙期/多拠点のうち2つ以上確認

### Aランク
- 上記のうち1つ確認。ただし業務量は未確認

### Bランク
- 業種・規模は合うが、入力痛みシグナルなし

### 除外
- 作業量が少ない
- 既存外注済み
- システム完結
- 入力対象が不明

## 7. 受付突破原則
受付では、システム導入提案に聞こえないようにする。

弱い言い方：

```txt
DX化や工数削減の事例共有でお電話しました。
```

強い言い方：

```txt
システム導入のご提案ではなく、紙の発注書・申込書・FAX・PDFなどを、社内で手入力されている業務が残っていないかの確認でお電話しました。
```

## 8. 本人接続後の確認順序
1. 紙・FAX・PDF・申込書・伝票などの入力業務が残っているか
2. 日常的にあるのか、月末・繁忙期だけ増えるのか
3. 誰が処理しているのか
4. 既存システムはあるのか
5. システムの前段入力だけ人が対応していないか
6. 外に出せる余地があるか

## 9. KPI

### 通常KPI
- 総架電数
- 着電数
- 受付突破数
- 本人接続数
- 受付拒否数
- 本人拒否数
- アポ数
- アポ率

### BPO専用KPI
- 具体業務確認数
- 入力物確認数
- 業務量確認数
- 既存システム確認数
- 外出し余地確認数
- 有効アポ数
- 有効アポ率
- 低確度アポ混入率

### 時間帯別KPI
時間帯別の架電分析では、着電率だけで判断しない。

見る順番：

```txt
電話数
↓
着電率
↓
対象者通話数 / 対象者通話率
↓
本人拒否率
↓
本人資料請求率
↓
アポイント率
↓
有効アポ率
```

危険な見誤り：

```txt
着電率が高い = 良い時間帯
```

ではない。

BPO・入力代行案件では、着電率が高くても対象者通話率が低い時間帯は、受付に捕まって本人へ回らない「受付処理場」になっている可能性がある。

そのため、時間帯別に以下を分ける。

| 分類 | 判断基準 | 対応 |
|---|---|---|
| 有効時間帯 | 対象者通話率が相対的に高い | 優先配分 |
| 検証時間帯 | 電話数は多いが対象者通話率が低い | トークABテスト |
| 削減時間帯 | 着電率は高いが対象者通話率が低い | 架電量削減・受付突破トーク改修 |
| 捨て時間帯 | 対象者通話がほぼ出ない | 原則停止 |

## 10. 共栄データセンター案件からの抽出
共栄データセンター案件では、過去分析で「アポは取れているが受注に繋がりにくい」構造が確認された。
原因は、アポ数ではなくアポ品質にある。

営業OS上の判断：

```txt
共栄DC = データ入力代行を説明する案件ではなく、
システム化しきれていない紙・FAX・PDF・申込書・伝票入力を外部に逃がす案件。
```

### 10.1 2026年6月・時間帯別本人通話分析からの追加学習
2026-06-01〜2026-06-30の時間帯別分析では、総架電1,371件に対し対象者通話103件、対象者通話率7.51%。
さらに、対象者通話103件すべてが本人拒否で、本人資料請求・アポイントは0件だった。

示唆：

```txt
時間帯最適化だけではアポは増えない。
本人接続後の第一声が弱ければ、接続が増えても拒否が増えるだけ。
```

特に16時台は着電率56.42%と高い一方、対象者通話率は4.13%と低い。
このような時間帯は「電話には出るが本人へ回らない時間帯」として扱う。

運用判断：

- 10時台・13〜14時台を優先検証する。
- 15〜16時台、特に16時台は削減または別トーク検証枠にする。
- 12時台は対象者通話が出ない限り優先度を下げる。
- 本人接続後は「DX」「工数削減」ではなく、紙・FAX・PDF・発注書・申込書・伝票・検査記録などの具体入力物確認から入る。

この型は、BPO・入力代行・AI-OCR・RPA・事務代行・DX支援案件へ横展開する。
