# PM / PM補佐 報告テンプレ

## 定例資料まとめ報告
```md
お疲れ様です。

定例資料について、架電KPI・本人拒否/受付拒否ログ・商談状況をもとに、現状の分析と次回改善方針を整理しました。

今回の主な論点は以下です。

1. 架電KPI上のボトルネック
2. 本人拒否/受付拒否の主な理由
3. 商談接続で詰まっている箇所
4. 次回台本・リスト・運用面の改善案

また、商談者の切り替わりにより一部スムーズに進まなかった点については、こちらでも運用面を確認し、今後は情報共有と役割分担がより明確になるよう改善します。

資料をご確認いただき、必要であれば次回定例で補足します。
```

## 先方/社内への責任分界点報告
```md
今回の件について、問題を以下のように切り分けています。

## 事象
- {何が起きたか}

## 確認できている事実
- {事実1}
- {事実2}

## 原因の切り分け
- リスト側：{問題有無}
- 架電運用側：{問題有無}
- 引継ぎ側：{問題有無}
- 資料/台本側：{問題有無}

## 改善
- {次回からの具体策}

## 対策
- {チェックルール/共有ルール/禁止設定など}
```

## 原則
- 個人攻撃にしない。
- 事実・原因・改善・対策を分ける。
- 「誰が悪い」ではなく「どの管理ポイントが抜けたか」で報告する。
