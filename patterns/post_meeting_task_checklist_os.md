# 商談後タスクチェックリストOS｜登録・送付・録画・報告漏れを防ぐ型

抽出元：株式会社アイネットテクノロジーズ案件  
更新日：2026-06-29

---

## 0. 位置付け

このOSは、商談後のSC登録、Mazrica登録、サンクスメール、リマインド、Teams報告、録画格納などの漏れを防ぐための横展開チェックリストである。

商談後作業は、担当者の記憶や善意で管理してはいけない。  
特に、Outlook / Teams / SalesCrowd / Mazrica / メール / Drive をまたぐ案件では、チェックリストなしでは事故が起きる。

---

## 1. 基本思想

```txt
商談は、終わった瞬間が一番危ない。

商談後の登録・送付・報告が漏れると、
商談成果そのものが消える。
```

商談後タスクの完了条件は「やったつもり」ではない。

```txt
登録済み
送信済み
送信予約済み
Teams報告済み
録画確認済み
次アクション登録済み
```

ここまで確認して完了とする。

## 2. 商談後チェックリスト

### 2.1 商談直後

| # | 項目 | 完了条件 |
|---:|---|---|
| 1 | 商談実施有無の確認 | 実施 / 不実施 / リスケを記録 |
| 2 | 商談メモ作成 | 温度感・課題・関与部署・次アクションを記録 |
| 3 | 次回アクション判定 | 提案 / 資料送付 / フォロー / 失注 / ナーチャリングを分類 |
| 4 | 録画確認 | 録画ファイルが存在し、音声が入っているか確認 |
| 5 | 音声不備時の補完 | 当日中に商談メモを詳細化 |

### 2.2 システム登録

| # | 項目 | 完了条件 |
|---:|---|---|
| 6 | SalesCrowd登録 | 商談結果・次アクションを登録 |
| 7 | Mazrica登録 | 取引先・コンタクト・案件・アクションを登録 |
| 8 | 次アクション登録 | 次回予定・フォロー期限・担当者を登録 |
| 9 | カレンダー反映 | Outlook / Google Calendar 等に正しく登録 |
| 10 | Teams報告 | クライアント側チャットに必要情報を報告 |

### 2.3 メール対応

| # | 項目 | 完了条件 |
|---:|---|---|
| 11 | サンクスメール作成 | 宛先・社名・氏名・日時・署名を確認 |
| 12 | サンクスメール送信 | 送信済み / 送信予約済みを確認 |
| 13 | リマインドメール作成 | 商談日時・会議URL・宛名・署名を確認 |
| 14 | リマインド送信予約 | 送信予約済みを確認 |
| 15 | 送信チェック | 誤字、コピペ残り、宛名違い、署名漏れを確認 |

## 3. Mazrica登録の考え方

Mazricaでは、案件を「登録して終わり」にしない。  
次アクション管理まで入れて初めて意味がある。

```txt
取引先
↓
コンタクト
↓
案件
↓
アクション
↓
完了アクション
↓
次アクション
```

### 3.1 コンタクト登録

同じ会社でも、別人物と関わる場合はコンタクトを追加する。

### 3.2 アクション登録

次回日程、フォロー予定、資料送付、リマインド、面談準備などをアクション化する。

### 3.3 完了アクション

商談実施、架電実施、メール送付などは完了アクションとして登録する。

## 4. 録画・音声チェック

### 商談前

```txt
録画方法を確認する
音声入力を確認する
画面共有の方法を確認する
録画開始を確認する
```

### 商談後

```txt
録画ファイルが生成されているか
音声が入っているか
ファイル名・格納先が分かるか
共有権限があるか
```

### 音声がない場合

```txt
当日中に商談メモを詳細化する
参加者に補足確認する
次回からの録画方法を修正する
```

---

## 5. よくある事故

| 事故 | 原因 | 防止策 |
|---|---|---|
| サンクスメール未送信 | 下書き保存で止まる | 送信済み/送信予約済みをチェック欄で確認 |
| リマインド漏れ | カレンダー登録だけで安心する | リマインド送信予約を別項目で管理 |
| 宛名ミス | テンプレ流用 | 社名・氏名・日時・署名の4点確認 |
| Mazrica未登録 | SC登録だけで完了と勘違い | CRM別に完了欄を分ける |
| 録画音声なし | 録画開始のみ確認 | 商談後に音声再生まで確認 |
| Teams報告漏れ | 内部登録だけで止まる | クライアントチャット報告を完了条件にする |

---

## 6. 運用ルール

1. 商談後タスクは1枚の管理表で見える化する。
2. 完了欄はチェックボックスまたはプルダウンにする。
3. 送信予約済みと送信済みは分ける。
4. 顧客向けメールは初動だけ上長確認を入れる。
5. ミスが発生したら、個人指摘ではなくチェックリスト項目を増やす。

---

## 7. 一言まとめ

```txt
商談後作業は営業成果の保全作業。

SC、Mazrica、Outlook、Teams、メール、録画を跨ぐ案件では、
チェックリストなしの運用は事故前提。
```
